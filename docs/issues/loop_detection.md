# 多入力対応ループ検知

## 現行仕様

`onidb.py loops` はAgentごとの理想定常運転を1つの不可分な反応式として扱う。
cycle当たりに正規化済みの `active / always / eat / harvest` を合算し、連続稼働、
即時収穫、毛刈り条件維持を仮定する。`death` は寿命で周期換算されていないため除外する。
電力を含むすべての消費資源について、反応集合全体の正味収支が0以上に
なる活動量の組み合わせを線形計画法で求める。

これにより、以前の「複数入力Agentを全除外」という制限を廃止した。
多入力の一部だけを使った偽ループは、残りの入力が負収支になるため不成立となる。

## 探索方法

1. 入力資源→出力資源の投影グラフを作る。
2. 強連結成分ごとにサイクル候補を列挙する。
3. 候補のAgentを元の多入力・多出力反応式に戻す。
4. 全資源の正味収支≥0、かつ少なくと1資源の正味収支>0をLPで検証する。
5. 包含関係にある候補は最小反応集合に整理する。

`--max-depth` は投影グラフ上の最大反応数。既定値8は、現在最長の既知ループである
サワーガスボイラーを含む。より長い候補は値を増やして監査できる。

## 回帰テスト

- 多入力の副原料が供給されない場合は検出しない。
- 複数入力と電力帰還を含む閉ループを検出する。
- 開始資源が収支0でも、別の資源が増えるループを検出する。
- 実DBで石油ボイラーとサワーガスボイラーを検出する。

## 追加データ

原油を402.85℃以上に加熱したときの `熱分解(原油→石油)` を質量比1:1で追加。
製油装置の50%変換とは別の、100%効率の自然熱変換である。
