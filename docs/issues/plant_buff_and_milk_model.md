# 植物バフ(農家の手・グラブグラブ世話)とモー乳産卵ブーストのモデリング

本ファイルが今回登録した数値の一次記録。seed.sqlのコメントは要約であり、
「wikiで調べた生の値 → DB登録値への換算式」の全量はここに残す。

## 出典

- 農家の手: https://oxygennotincluded.wiki.gg/wiki/Farm_Station
- グラブグラブ世話: https://oxygennotincluded.wiki.gg/wiki/Divergent
- モー乳: https://oxygennotincluded.wiki.gg/wiki/Brackene
- 繁殖速度と幸福度: https://oxygennotincluded.wiki.gg/wiki/Critter

## recipe命名規則

- バフ付き変種のrecipe名はバフ名そのもの。複数バフの重ねがけは『+』区切りで併記する
  (例: `農家の手`, `グラブグラブ世話`, `農家の手+グラブグラブ世話`)。
- 産卵の幸福度ブースト変種は `産卵(モー乳)`。
- onidb.pyのオプションゲート(後述)はrecipe名の部分一致で判定するため、
  ゲート対象バフ名は必ずこの表記でrecipe名に含めること。

## A-1. 農家の手(Farmer's Touch / 農業ステーション)

wikiで調べた生の値(wiki.gg "Farm Station"):

- 複製人間が肥料5kgを1回植物に与えると生産(成長)速度+100%(=生産×2)
- 効果の持続: 1 + 0.1×農家の農業スキル cycle、最大5cycle
- 対象外植物あり(下記リスト)

DB登録値への換算:

- 肥料コスト: 5kg/回 ÷ 5cycle(実用最大持続) = **肥料 -1 kg/cycle** をactive入力に追加
- 収穫レート: 既存harvestレート×2。灌漑等の既存入力は基礎recipeと同一に複製
- 常時有効な通常recipe変種(肥料という質量コストで結合されるためソルバー的に安全)

### 農家の手 対象外植物(seed.sqlに存在するもの、wiki.gg除外リストより)

- ミールウッド (Mealwood)
- ウィーズウォート (Wheezewort)
- オキシファーン (Oxyfern)
- ほか植木鉢・壁掛け鉢など農業タイル以外に植えた個体全般

これらには農家の手系recipe変種を作らない(組合せ変種も同様)。

## A-2. グラブグラブ世話(Grubgrub Rub)

wikiで調べた生の値(wiki.gg "Divergent"):

- グラブグラブが居れば対象植物の生産速度+50%(=生産×1.5)。質量コストなし
- 満腹個体で約8株/cycle程度を世話可能(本DBでは株数は管理せず「居るか居ないか」のみ)
- 対象は地上・農業タイルの植物(植木鉢は対象外)

DB登録値への換算:

- 収穫レート: 既存harvestレート×1.5。入力は基礎recipeと同一、追加質量コストなし
- 両バフ重ねがけ(加算): +100%+50% = +150% → 収穫×2.5、肥料 -1 kg/cycle

### オプションゲートの理由

グラブグラブ世話recipeは入力が基礎recipeと同一で出力だけ1.5倍の
「質量結合なしの上位互換」なので、常時有効にするとLPソルバーが無条件に
基礎recipeより優先して選び、グラブグラブの存在という前提が収支に反映されない
偽経路になる(direct_grazing_resource_alias.md の偽経路問題と同型)。
そのため onidb.py の `OPTIONAL_BUFF_RECIPES = ("グラブグラブ世話",)` により
recipe名に『グラブグラブ世話』を含む行を load_reactions() で既定除外し、
`--buff グラブグラブ世話`(calories / loops サブコマンド)指定時のみ含める。
『農家の手+グラブグラブ世話』も部分一致で同様にゲートされる。

## B. 給餌+モー乳 — 幸福度による産卵ペースのブースト

**2026-07-03 設計変更(ユーザー指摘)**: 産卵は「餌を食べている個体」が大前提のため、独立recipe
『産卵』『産卵(モー乳)』は廃止した。産卵(Groomedレートの卵)は各**給餌recipe**の出力に統合し、
モー乳ブーストは各給餌recipeの変種 `'<餌名>+モー乳'`(給餌IO+モー乳-5+卵×2.125)として登録する。
給餌と産卵が不可分になったことで、餌なしで卵だけが湧く偽経路がループ探索に混入しなくなり、
産卵反応がソルバーに給餌コスト込みで参加する。以下の数値・導出はそのまま有効。

wikiで調べた生の値:

- 繁殖速度 = 基礎値 × (100% + 225%×幸福度)(wiki.gg "Critter")
- Groomed(給餌+毛づくろい済み)の公表産卵周期は幸福度4相当 = 基礎の1000%
  (100%+225%×4 = 1000%。野生90cycle → Groomed 9cycleの10倍と整合)
- モー乳(Brackene): クリッターファウンテンで1回5kg固定、+5幸福度、持続1cycle
  (複製人間のスキル非依存)(wiki.gg "Brackene")

DB登録値への換算:

- Groomed+モー乳 = 幸福度9 → 100%+225%×9 = 2125% = 基礎の2125%
- 既存の産卵recipeレートはGroomed(1000%)基準なので、倍率 = 2125%/1000% = **×2.125**
- コスト: **モー乳 -5 kg/cycle**(5kg/回 × 1回/cycle、持続1cycleのため)
- この一般式は terms「繁殖速度と幸福度」にも登録済み

モー乳の供給元: ぷすぷすモー(搾乳ステーション)、登録済み
(植物の殻 -200 + 天然ガス +10 → モー乳 +50 kg/cycle)。新規登録は不要だった。

### 既知の単純化

- (解消済み)かつては産卵が独立recipeで給餌と結合されていなかったが、上記の設計変更で
  各給餌recipeへ統合済み。
- '+モー乳'変種recipeは常時有効(モー乳5kg/cycleという質量コストで結合されるため
  オプションゲート不要)。
- グラブグラブ世話は株数(約8株/cycle/匹)を管理せず、有効/無効の二値。
- 農家の手の持続は農業スキル最大(5cycle)前提。スキルが低いと肥料コストは
  1kg/cycleより大きくなる。
